# BrandGuard — Decentralized Marketing Escrow for Influencers

BrandGuard is a decentralized marketing escrow contract that holds a brand's sponsorship funds and only releases payment to the influencer when their published content qualifies against the agreed sponsor guidelines and tone guidelines, evaluated automatically by GenLayer AI Validators.

### Why BrandGuard DIES without GenLayer
> **Traditional blockchains (like Ethereum or Solana) cannot natively access web page contents or execute non-deterministic natural language audits, meaning BrandGuard would die without GenLayer's unique ability to scrape published content URLs and run decentralized, semantic consensus checks to verify qualitative marketing rules on-chain.**

---

## Features
- **Qualitative Web Scraping**: The AI Leader node calls `web.render` on the influencer's post, scrapes the text, and feeds it into the LLM prompt.
- **AI Brand Manager**: Evaluates subjective parameters like tone, naturalness, enthusiasm, guideline compliance, and checks for banned words case-insensitively.
- **Semantic Consensus**: A custom `validator_fn` parses outputs and verifies that all validators agree on the final transaction verdict (`RELEASE` or `REJECT`) rather than comparing raw strings.
- **Interactive Revision Loop**: Rejections put the campaign into a `REJECTED` state, allowing the influencer to edit their post and resubmit without needing a brand-new campaign escrow contract.
- **Brand Refund Capability**: Sponsors can cancel the escrow and withdraw their GEN funds if the campaign has not been submitted or completed yet.

## 1. How to Deploy the Intelligent Contract

To deploy your contract using **GenLayer Studio**:

1. Open **[GenLayer Studio](https://studio.genlayer.com/)**.
2. Create or upload a new file named `brandguard.py` containing the code in `contracts/brandguard.py`.
3. Check the compiler detection header at the top:
   ```python
   # v0.2.16
   # { "Depends": "py-genlayer:1jb45aa8ynh2a9c9xn3b7qqh8sm5q93hwfp7jqmwsfhh8jpz09h6" }
   ```
4. Compile the file and select `Contract` in the compilation output panel.
5. Click **Deploy**.
6. Once deployed successfully, copy the **Contract Address** generated by GenLayer Studio.

## 2. How to Run the Frontend

### Prerequisites
Make sure you have **Node.js** (v18 or higher) and **npm** installed on your machine.

### Installation Steps

1. Navigate to the `frontend` folder:
   ```bash
   cd D:\Gen\BrandGuard\frontend
   ```

2. Install the dependencies:
   ```bash
   npm install
   ```

3. Configure your contract address. Open the `D:\Gen\BrandGuard\frontend\.env` file and paste your deployed contract address:
   ```env
   VITE_CONTRACT_ADDRESS=0xYourDeployedContractAddressHere
   ```

4. Start the local Vite development server:
   ```bash
   npm run dev
   ```

5. Open the local port in your browser (usually `http://localhost:3000`).

## 3. How to Use the dApp

1. **Connect Account**: 
   Click **Connect Wallet**. The app will connect to your browser extension or generate a secure fallback local account.
2. **Sponsor Deploys Escrow (Brand Role)**:
   Enter the Influencer's wallet address.
   Specify the GEN payout amount to lock.
   Provide sponsor rules (e.g. "sound natural, enthusiastic, mention the new flavor") and comma-separated banned words (e.g. "advertisement, cheap").
   Click **Lock Payout Escrow** to deposit and deploy the campaign.
3. **Submit URL (Influencer Role)**:
   Connect as the influencer. Find the active campaign under the dashboard, paste the URL of your published post, and click **Submit**.
4. **Trigger AI Review (Brand / Influencer)**:
   The campaign status becomes `SUBMITTED`. Click **Run AI Brand Manager Audit** to trigger the GenLayer validator network.
5. **Winnings / Revision**:
   *   If approved, the contract automatically releases the escrowed funds and pays the influencer wallet directly.
   *   If rejected, the influencer can read the AI critique reasoning box, edit their post, and resubmit using the same campaign dashboard.
